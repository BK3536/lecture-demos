# Deploy Boundaries

## Main rule
- Keep lecture authoring assets, generated figures/pages, and deployment assumptions connected.
- Do not treat deployed HTML as the sole source of truth when source scripts generate the visuals.
- `labs.remilab.ai` static content is outside this repo now; do not treat `teaching/lecture/` as the deploy surface for labs.

## Subproject caution
- If work targets the sibling repo `../grad-radar-ai/`, defer to its own deploy rules there.
